This website originally used WordPress, a popular content management system used by many bloggers and web developers on many websites and blogs across the internet. WordPress is excellent for learning web developers who want a website that is easy to set up. It is also great for anyone who doesn't have time to build a website from scratch and doesn't really need to. However, it isn't perfect.

WordPress contains a lot of functionality that I don't use, and a lot of this functionality is implemented in an inefficient manner; this slows the website down for myself, as well as for visitors. This unnecessary functionality can also make websites difficult to customise and maintain. I noticed this a lot when I was trying to make a custom WordPress theme for it in 2012; while I managed to get something together in the end, the code was a complete mess and I couldn't get certain aspects of my theme the way I wanted them.

I've wanted to rebuild my website for about a year now. I'm sure I could have fixed most of my problems with WordPress, but I still wouldn't have had full control over my website. By building my own website, I was able to build my own functions, design the database and even choose what server-side programming languages I wanted to use.

The main reason I didn't do this rebuild earlier is because I simply didn't know how to. Although I've known HTML and CSS for years, I've only known the basics of a couple of interpreted programming languages and databases. Some work on my computing course inspired me to experiment and learn more, but I recently took a web development evening class and got a kick start with learning how to access a MySQL database with PHP. After that, I was ready to get started.
